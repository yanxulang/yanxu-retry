# 言韧

言韧（`yanxu-retry`）1.0.0 是面向言序 1.1.6+ 的确定性重试、退避与断路器库。它把策略、执行控制和故障状态分离：生产环境可真实等待，测试与外部调度器可注入等待、观察、停止和单调时间，同时复用完全相同的尝试与状态逻辑。

## 安装

```sh
yanbao add 言韧 --git https://github.com/yanxulang/yanxu-retry.git --version '^1.0'
```

等价清单：

```toml
[依赖]
言韧 = { git = "https://github.com/yanxulang/yanxu-retry.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

言韧没有第三方依赖，也不申请文件、网络、监听、环境、进程或原生扩展权限。

## 快速开始

```yanxu
引「包:言韧」为 言韧；

法 请求一次（次数：数）：文 则
    若 （次数 小于 3） 则
        抛 「YANQIU_TIMEOUT：请求超时」；
    终
    归 「响应成功」；
终

定 策略 为 言韧.重试策略（4，0.1，2，1）.设抖动（0.5，2026）；
定 可恢复：法 为 言韧.可重试代码（【「YANQIU_TIMEOUT」】）；

言 言韧.执行（策略，请求一次，可恢复）；
```

`最大次数`包含首次调用；只有确定为暂时性且可安全重放的错误才应重试。写操作应使用幂等键、事务或去重协议，不能用`总是重试`替代业务判断。

## 可观察执行

`执行控制`可替换真实等待、接收生命周期事件，并在每次尝试前停止：

```yanxu
法 外部等待（秒：数，上下文：典）：空 则
    # 交给任务调度器；测试中也可只记录秒数
    归 空；
终

法 观察（事件：典）：空 则
    言 事件【「阶段」】；
    归 空；
终

定 控制 为 言韧.执行控制（）
    .设等待器（外部等待）
    .设观察器（观察）；

定 结果 为 言韧.受控执行（策略，请求一次，可恢复，控制）；
```

观察阶段为`开始`、`失败`、`等待`、`成功`、`放弃`、`耗尽`或`停止`。回调错误有独立稳定代码，不会被误当成业务错误重试。

## 断路器

```yanxu
定 所断路器 为 言韧.断路器（5，30000）；

法 调用上游（）：文 则
    归 「上游结果」；
终

言 所断路器.调用（调用上游）；
```

断路器提供关闭、开启、半开三态。`调用`会取得许可并自动记录结果；半开状态只允许一个在途探针。手工编排时用`获取许可`，并保证随后调用`记成功`或`记失败`。`快照`提供连续失败、冷却剩余和成功、失败、拒绝、开启计数。

重试与断路器可直接组合：

```yanxu
定 值 为 言韧.受控执行经断路器（策略，所断路器，请求一次，可恢复，控制）；
```

## 安全边界

| 项目 | 硬上限 |
| --- | ---: |
| 尝试次数 | 4096 |
| 单次延迟 | 86400 秒 |
| 计划累计等待 | 604800 秒（7 天） |
| 退避倍率 | 1000 |
| 错误代码白名单 | 64 |
| 失败阈值 | 4096 |
| 恢复时间 | 604800000 毫秒（7 天） |

完整退避表按尝试数线性生成；抖动由显式整数种子确定，可跨树解释器与字节码 VM 复现。
