# 从 0.1 迁移到 1.0

## 依赖

把依赖范围和修订改为稳定标签：

```toml
[依赖]
言韧 = { git = "https://github.com/yanxulang/yanxu-retry.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

最低言序仍为 1.1.6，且没有新增第三方依赖或权限。

## 保留的 0.1 入口

- 四参数`重试策略`及四个公开只读基础字段；
- `默认策略`和`立即策略`；
- `延迟秒`和`延迟表`；
- `总是重试`和`从不重试`；
- `执行`、`立即执行`和四参数`尝试执行`；
- 两参数`断路器`、`状态`、`是否允许`、`记成功`、`记失败`、`失败数`和`调用`。

在有效 0.1 参数范围内，未启用抖动的退避数值保持一致。

## 行为变化

### 策略边界

最大次数现在必须是 1 至 4096 的安全整数。单次延迟从 0.1 的 60 秒上限扩展到 86400 秒，但完整无抖动计划不得超过七天；倍率上限从 100 扩展到 1000。

非法配置不再统一返回`YANREN_POLICY`，而是按次数、延迟、倍率、预算、抖动和种子返回细分稳定代码。依赖完整错误消息的测试应改用`错误详情`中的`代码`。

### 延迟位置

`延迟秒(策略, 已失败次数)`现在只接受 1 至`最大次数 - 1`。0.1 虽允许查询计划外位置，但执行器永远不会使用它们；迁移时改用更大的策略或只查询`延迟表`。

退避实现改为封顶迭代和线性完整计划，避免幂溢出与二次生成成本。

### 尝试结果

`尝试执行`现在成功和失败都含固定字段：

```text
成功、值、错误、次数
```

成功时`错误 = 空`，失败时`值 = 空`。0.1 中只在对应分支出现的字段现在始终存在。读取旧字段的代码继续工作；依赖完整典键集合的测试需更新。

失败错误使用`错误详情`，可识别组织约定的`YAN…：`代码，而不只是运行时的源错误字段。

### 回调隔离

判定器错误现在转换为`YANREN_PREDICATE`，不再偶然替换业务错误后被外层误判。新增观察器、等待器和停止判定也各自拥有稳定错误代码。成功观察器失败不会重放已经成功的业务操作。

### 断路许可

`是否允许`明确为只读查看，不占用半开探针。新增`获取许可`用于手工流程；`调用`内部使用它，并保证一个同步共享实例只有一个半开探针。

冷却期间额外失败不再重置开启时刻。`记成功`仍可人工关闭，但会计入成功总数；只需清状态而不计成功时使用`重置`。

### 时间

断路器可在关闭状态注入时间源，并拒绝负数、非安全整数或倒退。依赖修改系统时间推进冷却的测试应改为虚拟时间源和自定义等待器。

## 新能力

- 显式种子的可复现向下抖动；
- `基础延迟秒`、`总延迟秒`和结构化`计划表`；
- `可重试代码`白名单；
- 可注入等待、观察和停止的`执行控制`；
- 事件模式 v1 与固定阶段；
- 受控和断路器组合执行入口；
- 断路器许可、剩余冷却、快照、重置和累计计数；
- 结构化错误详情与完整错误目录。

## 迁移检查

1. 搜索`总是重试`，确认操作幂等且错误域有限。
2. 检查是否查询了计划外的`延迟秒`位置。
3. 更新依赖完整`尝试执行`典相等的测试。
4. 把取消和总截止时间接入`设停止判定`。
5. 手工断路调用改用`获取许可`并保证记录结果，或直接用`调用`。
6. 用固定种子和虚拟时间替换真实睡眠测试。
7. 为`放弃`、`耗尽`、`停止`和断路拒绝建立监控。
8. 验证重试层与代理、消息系统、数据库驱动自身的重试没有叠加放大。
