# 可观察性契约

## 重试事件模式 v1

观察器接收一个新建的典：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `模式版本` | 数 | 固定为`事件模式版本`，1 |
| `阶段` | 文 | 生命周期阶段 |
| `次数` | 数 | 当前尝试；`停止`时为已完成次数 |
| `最大次数` | 数 | 策略允许的总尝试数 |
| `累计等待秒` | 数 | 到该阶段已调度的逻辑等待总量 |
| `等待秒` | 数 | 仅`等待`阶段可能非零 |
| `错误` | 典或空 | 最近一次结构化错误 |

`错误`与`错误详情`形状一致：`代码`、`消息`、`源代码`、`类别`、`位置`、`踪迹`。

## 阶段

| 阶段 | 发送时机 | 错误 |
| --- | --- | --- |
| `开始` | 业务操作调用前 | 上一次错误或空 |
| `失败` | 本次业务操作抛错后 | 本次错误 |
| `等待` | 判定允许重试、等待器调用前 | 本次错误 |
| `成功` | 业务操作返回后 | 空 |
| `放弃` | 判定器返回假 | 本次错误 |
| `耗尽` | 最后一次业务操作失败 | 本次错误 |
| `停止` | 停止判定在下一次尝试前返回真 | 最近错误或空 |

停止判定还会收到`阶段 = 检查`的上下文。它不是观察事件，`次数`表示下一次尝试编号。首次检查的次数为 1、错误为空、累计等待为 0。

## 顺序保证

单次同步执行内，事件按程序顺序发送，不并发调用观察器。成功尝试严格是`开始 → 成功`；可重试失败严格是`开始 → 失败 → 等待`；终止失败在`失败`后追加`放弃`或`耗尽`。

若观察器抛错，当前事件之后不再发送事件，执行以`YANREN_OBSERVER`结束。观察器错误不触发业务重试。

## 指标映射

推荐从事件派生：

- 尝试计数：`开始`；
- 业务失败计数：`失败`，按`错误.代码`分组；
- 重试调度计数与计划等待：`等待`和`等待秒`；
- 最终成功：`成功`；
- 不可重试终止：`放弃`；
- 次数耗尽：`耗尽`；
- 截止或取消：`停止`。

不要把`失败`直接当作最终失败；一次执行可能随后成功。不要把`累计等待秒`当作墙钟耗时，自定义等待器可能跳过、批处理或异步调度。

## 断路器快照

`快照()`返回：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `状态` | 文 | 关闭、开启或半开 |
| `连续失败` | 数 | 当前连续失败数 |
| `失败阈值` | 数 | 固定配置 |
| `恢复毫秒` | 数 | 固定配置 |
| `剩余恢复毫秒` | 数 | 开启时剩余冷却，否则 0 |
| `半开探针占用` | 理 | 是否已有半开许可在途 |
| `成功总数` | 数 | 调用`记成功`的累计次数 |
| `失败总数` | 数 | 调用`记失败`的累计次数 |
| `拒绝总数` | 数 | `获取许可`拒绝的累计次数 |
| `开启总数` | 数 | 首次开启和半开重开的累计次数 |

读取快照可能推进已过冷却期的状态。累计计数不因`重置`清零，适合进程生命周期指标；应用需要区间值时应保存上次快照做差。

## 标签基数

指标标签应使用有限集合：库名、依赖名、稳定错误代码和阶段。不要把完整错误消息、踪迹、URL、用户标识、种子或请求正文放入标签。错误详情可能含敏感实现信息，应在日志或追踪层按策略过滤。
