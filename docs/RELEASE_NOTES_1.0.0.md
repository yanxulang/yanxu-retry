# 言韧 1.0.0

言韧 1.0.0 把首版指数退避与三态断路器推进为具有稳定错误、确定性时间、可观察执行和资源预算的基础库，同时保留 0.1 的主要入口。

## 主要内容

- 四参数策略保持兼容，并增加 4096 次尝试、单次一天、总计划七天和倍率 1000 的硬边界。
- 封顶迭代替代幂运算；完整延迟表和计划表按尝试数线性生成。
- 显式整数种子提供可复现的部分或完全抖动。
- `可重试代码`从组织约定的`YAN…：`错误构造隔离白名单。
- `执行控制`注入等待器、观察器和停止判定，事件模式 v1 覆盖完整生命周期。
- 结构化尝试结果固定为`成功/值/错误/次数`。
- 回调错误与业务错误隔离，成功观察失败不会重放业务操作。
- 断路器支持单调时间源、只读许可查看、唯一半开探针、剩余冷却、人工重置和累计快照。
- 重试与断路器提供直接组合入口，共享虚拟时间可确定验证完整恢复路径。

## 安全与运行边界

- 无第三方包依赖。
- 不申请文件、网络、监听、环境、进程或原生扩展权限。
- 业务错误在直接执行入口原样传播；库错误使用`YANREN_`稳定前缀。
- 重试是同步且协作式停止，不抢占正在执行的业务操作。
- 断路器是进程内同步状态，不提供跨线程、跨进程或分布式原子协调。

## 验证资产

- 6 份规格覆盖退避、抖动、最大边界、执行事件、停止、回调隔离、断路三态、唯一半开许可、时钟异常和组合恢复。
- 3 个示例覆盖错误白名单、可观察外部调度和虚拟时间断路恢复。
- 1 个确定性负载文卷覆盖 240 个计划项、256 次业务尝试和 128 组断路状态更新。
- 规格、示例和基准在言序 1.1.6 与 1.1.8 中分别通过树解释器/字节码 VM 对照。
- Release 门禁覆盖 Linux、macOS、Windows、锁图、生成 API、包权限和 Release YXB。

## 兼容提示

- 计划外`延迟秒`查询现在报`YANREN_ATTEMPT`。
- 无效策略错误从统一代码拆分为精确代码。
- `尝试执行`结果新增固定的空分支字段。
- 半开手工调用应使用`获取许可`，查看状态仍用`是否允许`。
- 开启期间额外失败不延长当前冷却。
