# 兼容性

## 言序

言韧 1.0 的最低言序版本是 1.1.6。源码、规格、示例、基准、锁图和 Release YXB 同时在最低版本与当前稳定线 1.1.8 验证。

| 能力 | 1.1.6 | 1.1.8 |
| --- | --- | --- |
| 树解释器 | 支持 | 支持 |
| 字节码 VM | 支持 | 支持 |
| 树/VM 对照 | 支持 | 支持 |
| 生成 API | 读取兼容 | 生成门禁 |
| Release YXB | 支持 | 支持 |

包清单声明`言序 = ">=1.1.6"`。1.0 不使用 1.1.7 或 1.1.8 才新增的语言语法。

## 操作系统

言韧只使用数学、随机、正则和时间标准库，不依赖平台路径、网络栈或原生扩展。发布 CI 在 Linux、macOS、Windows 上使用言序 1.1.6 执行静态检查、规格、示例、基准、树/VM 对照、锁图和 Release 构建；当前稳定言序执行完整质量门禁。

真实等待与系统毫秒值受宿主调度精度和系统时钟影响。需要跨平台精确可重放时，应注入等待器和单调虚拟时间源，不应断言真实睡眠的精确耗时。

## 依赖与权限

言韧 1.0 没有第三方包依赖。包清单拒绝文件、网络、TCP 监听、UDP 绑定、环境、进程和原生扩展能力。调用方传入的操作或回调仍在顶层应用自身的权限范围内执行。

## 稳定范围

1.x 保证：

- 四参数`重试策略`和公开只读基础字段；
- 从 1 开始的尝试编号，以及最大次数包含首次调用；
- 退避、全抖动和资源上限的数学语义；
- 生命周期事件模式 v1 的字段与阶段名；
- `尝试执行`结果的`成功/值/错误/次数`固定形状；
- 关闭、开启、半开状态名和单个半开许可语义；
- 断路器快照字段；
- `YANREN_`库错误前缀和文档列出的稳定错误代码。

系统错误的具体文字、错误踪迹布局、真实等待精度、正则引擎内部诊断和典的调试显示顺序不作为机器协议。

## 并发模型

重试执行和断路器方法都是同步调用。单个断路器实例在言序同步执行流内保证一个半开探针；它不提供跨线程、跨进程或分布式原子协调。多执行器共享时，应用须串行访问或在外层提供同步。集群级熔断应使用共享控制平面，不能依赖进程内实例。
